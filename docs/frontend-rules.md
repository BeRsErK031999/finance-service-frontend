# Frontend Rules

## Общие правила

- Изменения должны быть минимальными и целевыми
- Не делать лишний рефакторинг без запроса
- Не менять архитектуру без необходимости
- Сначала смотреть `README.md`, `AGENTS.md` и релевантные документы из `docs/`

---

## Что нельзя делать

- Не складывать бизнес-логику в pages
- Не вызывать backend напрямую из случайных компонентов
- Не использовать Redux для server-state
- Не строить generic abstraction раньше времени
- Не делать самостоятельные вычисления backend-статусов как source of truth

---
